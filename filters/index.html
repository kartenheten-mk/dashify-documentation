    Filters & Functions

    Dashify provides a powerful suite of filtering options to refine data visualization and analysis. Filters can be applied globally (Time, Numeric, Atlas) or locally (Column, Spatial).

    Global Filters

    These filters are defined in the dataSource configuration and affect all widgets in the dashboard.

    Time Filter (timeFilter)

    Creates an interactive date range slider.

    Options

    Property Type Default Description
    dateField string - The date field to filter on.
    defaultFromDays number - Default "from" days ago (older boundary).
    defaultToDays number - Default "to" days ago (newer boundary).
    minDays number - Minimum allowed days on the slider.
    maxDays number - Maximum allowed days on the slider.

    Configuration:

    "timeFilter": {
  "dateField": "created_date", // Date column in your data
  "defaultFromDays": 30, // Default start (days ago)
  "defaultToDays": 0, // Default end (days ago)
  "minDays": 0, // Minimum value on slider
  "maxDays": 365 // Maximum value on slider
}

    Numeric Filter (numericFilter)

    Creates a slider for filtering by a numeric attribute (e.g., age, price).

    Options

    Property Type Default Description
    field string - The numeric field to filter on.
    label string - Display label for the slider.
    min number - Absolute minimum value.
    max number - Absolute maximum value.
    defaultMin number - Default minimum selected value.
    defaultMax number - Default maximum selected value.
    step number 1 Step increment for the slider.
    unit string - Unit suffix (e.g., "$").
    presets object[] - Array of preset range buttons.

    Configuration:

    "numericFilter": {
  "field": "price", // Numeric column
  "label": "Price", // Label
  "defaultMin": 100, // Default minimum
  "defaultMax": 500, // Default maximum
  "min": 0, // Absolute minimum
  "max": 1000, // Absolute maximum
  "step": 10, // Slider step
  "unit": "$", // Value unit
  "presets": [ // Optional preset buttons
    { "label": "Cheap", "min": 0, "max": 100 },
    { "label": "Expensive", "min": 500, "max": 1000 }
  ]
}

    Atlas Filter (atlasFilter)

    A dropdown menu for selecting predefined geographic areas (e.g., municipalities, districts). Supports single or multiple sources.

    Options (Single Source)

    Property Type Default Description
    source string - Reference to a shared data source key.
    typeName string - WFS layer name containing filter geometries.
    titleField string - Field to display as the area title.
    geometryField string 'geom' Geometry field name.
    defaultSelected string[] - Array of titles to pre-select.
    sources object[] - For multi-source configuration (see Atlas Filter Details).

    Single Source Configuration (Simple):

    Use this format for a single layer of atlas areas.

    "atlasFilter": {
  "source": "geoserver1", // Optional: WFS Source key (defaults to dashboard source)
  "typeName": "admin:municipalities", // Layer name containing the areas
  "titleField": "name", // Field to display in the dropdown
  "geometryField": "geom", // Geometry field name (default: "geom")
  "defaultSelected": ["Stockholm"] // Optional: Pre-selected values
}

    Multi-Source Configuration (Advanced):

    Use this format to combine multiple layers (e.g., Districts and Municipalities) or group areas. See the Atlas Filter Guide for full documentation.

    "atlasFilter": {
  "sources": [
    {
      "label": "Municipalities", // Group label
      "source": "geoserver1", // WFS Source key
      "typeName": "admin:municipalities", // Layer name
      "titleField": "name", // Display field
      "geometryField": "geom", // Geometry field
      "defaultSelected": ["New York"], // Pre-selected values
      "selectAll": true // Enable "Select All" option
    },
    {
      "label": "Districts",
      "source": "geoserver1",
      "typeName": "admin:districts",
      "titleField": "district_name",
      "groupBy": "city_id" // Group by attribute
    }
  ]
}

    Local Filters

    These filters are applied directly in the user interface.

    Spatial Filter (Map Drawing)

    Users can draw polygons or rectangles on the map to filter features spatially.

    Usage: 1. Select the Draw Polygon or Draw Rectangle tool from the map controls. 2. Draw a shape on the map. 3. Only features intersecting the shape will remain visible. 4. Click the Trash icon to clear the filter.

    Column Filter (Table)

    Users can filter table data by specific column values.

    Usage: 1. Click the filter icon in a table column header. 2. Select values from the dropdown list. 3. Or type in the search box to filter across all columns.

    Helper Functions

    CQL Filters (cqlFilter)

    Apply complex logic using OGC Common Query Language (CQL). This filter is sent directly to the WFS server.

    Examples: * status = 'active' * population > 1000 AND category IN ('A', 'B') * BBOX(the_geom, -180, -90, 180, 90)

    Dynamic Date Expressions

    Use special placeholders in your CQL filters to represent dynamic dates. These are evaluated before the request is sent.

    Expression Description Example Output
    {{date:now}} Current timestamp 2023-10-27T10:00:00Z
    {{date:now-7d}} 7 days ago 2023-10-20T10:00:00Z
    {{date:now-1m}} 1 month ago 2023-09-27T10:00:00Z
    {{date:now-1y}} 1 year ago 2022-10-27T10:00:00Z
    {{date:startOfDay}} Start of today (00:00) 2023-10-27T00:00:00Z
    {{date:startOfWeek}} Start of current week 2023-10-23T00:00:00Z

    Example Usage:

    "cqlFilter": "created_date > '{{date:now-7d}}' AND status = 'open'"
