    Configuration

    Dashify is entirely configured via a JSON file located at public/config/config.json. This file defines the application's appearance, available data sources, map base layers, and the structure of each dashboard.

    File Structure

    The configuration file is a JSON object with the following top-level keys:

    • app: Global application settings (name, logo).
    • mapSources: Definitions for map base layers (WMS, Tile).
    • dataSources: Shared WFS data source definitions.
    • dashboards: An array of dashboard configurations.

    Example Structure

    {
  "app": {
    "name": "My Dashboard App",
    "logo": "/logo.png"
  },
  "mapSources": {
    "osm": {
      "type": "tile",
      "url": "https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png",
      "attribution": "&copy; OpenStreetMap contributors"
    }
  },
  "dataSources": {
    "geoserver1": {
      "name": "Main GeoServer",
      "wfsEndpoint": "https://geoserver.example.com/geoserver/ows",
      "version": "1.0.0",
      "outputFormat": "application/json"
    }
  },
  "dashboards": [
    // Dashboard configurations...
  ]
}

    Global Settings (app)

    Customize the application name and logo appearing in the sidebar and browser tab.

    Property Type Description Required Default
    name string The name of your application. Displayed in the sidebar header and browser title. Yes "Dashboard"
    logo string Path to the logo image (relative to public/). Used as favicon and sidebar icon. No Vite Logo
    welcomePage object Configuration for the welcome/home page. No -

    Welcome Page Configuration (welcomePage)

    Property Type Description
    title string Main title on the welcome page.
    subtitle string Subtitle text.
    showGettingStarted boolean Whether to show the "Getting Started" section.
    gettingStartedTitle string Title for the "Getting Started" section.
    gettingStartedItems string[] List of instruction items.

    Map Sources (mapSources)

    Define the base maps available for use in map widgets. You can define multiple sources (WMS or Tile) and reference them by key in your dashboard configuration.

    Tile Layer

    "osm": {
  "type": "tile",
  "url": "https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png",
  "attribution": "&copy; OpenStreetMap contributors"
}

    WMS Layer

    "topowebb": {
  "type": "wms",
  "url": "https://kartor.example.se/geoserver/wms",
  "version": "1.1.1", // or "1.3.0"
  "format": "image/png",
  "transparent": true,
  "defaultLayers": ["layer1", "layer2"]
}

    Shared Data Sources (dataSources)

    Define WFS endpoints that can be reused across multiple dashboards. This avoids repetition and makes maintenance easier.

    Property Type Description Required
    name string Display name for the data source. Yes
    wfsEndpoint string The URL of the WFS service. Yes
    version string WFS version (e.g., "1.0.0", "1.1.0", "2.0.0"). No (Default: "1.0.0")
    outputFormat string Response format (e.g., "application/json"). No (Default: "application/json")
    refreshInterval number Data refresh interval in seconds. No
    maxFeatures number Maximum number of features to fetch. No
    geometryField string Geometry field name (default: "geom"). No
    useUtcDates boolean Treat dates as UTC (default: false). No
    timeFilter object Time filter configuration (see Filters). No
    numericFilter object Numeric filter configuration (see Filters). No
    atlasFilter object Atlas filter configuration (see Filters). No

    Usage in Dashboards

    Once defined, you can reference a shared source in a dashboard configuration using the source property:

    "dataSource": {
  "source": "geoserver1", // References the key in "dataSources"
  "typeName": "workspace:layer_name",
  "cqlFilter": "status='active'"
}

    Syntax Rules

    • Format: The file must be valid JSON.
    • Comments: Standard JSON does not support comments. Do not include // or /* */.
    • Keys: All property keys must be enclosed in double quotes.
    • Strings: All string values must be enclosed in double quotes.
    • Booleans: Use true or false (no quotes).
    • Numbers: Use plain numbers (no quotes).
